- [研究称 AI 编程无法承担长期任务](https://www.v2ex.com/t/1199231) (V2EX)
  阿里巴巴与中山大学联合发表论文，通过消耗 100 亿 token 的实验证明，目前的 AI 编程模型在处理大型项目的长期任务时仍存在显著局限。这表明 AI 目前更适合作为提升效率的辅助工具，尚无法完全接管复杂的工程化开发，开发者仍需保持对业务逻辑的掌控。

- [微盟集团 AI 收入破亿，SaaS 商业化找到新路径](https://36kr.com/p/3728689823006343) (36kr)

- [AI 编码引发“劣币驱逐良币”争议](https://www.v2ex.com/t/1199158) (V2EX)

- [超算中心被黑事件后续：机密信息泄露](https://www.v2ex.com/t/1199198) (V2EX)

## 3. 工具与深读

- [Cook: Claude Code 编排工具](https://rjcorwin.github.io/cook/) (Hacker News)
  这是一个简单的命令行工具（CLI），旨在帮助用户更便捷地编排和使用 Claude Code。对于需要频繁与 AI 模型交互的开发者来说，此类工具能有效提升工作流自动化程度，是 AI 辅助编程工具链中的实用补充。

- [Nvidia Greenboost: 透明扩展 GPU 显存工具](https://gitlab.com/IsolatedOctopi/nvidia_greenboost) (Hacker News)
  该工具允许透明地利用系统内存（RAM）或 NVMe 固态硬盘来扩展 GPU 的显存（VRAM）。在显存日益成为 AI 推理和训练瓶颈的当下，这一开源方案为硬件资源受限的开发者提供了一种低成本的缓解思路。

- [RX: 一种新的随机存取 JSON 替代方案](https://github.com/creationix/rx) (Hacker News)
  RX 是一种旨在替代 JSON 的新型数据格式，其核心特性是支持随机存取。这意味着在处理大型数据文件时，无需像 JSON 那样解析整个文件即可读取特定数据，对于需要高效处理大规模数据集的应用场景具有重要价值。

- [Rob Pike 的编程规则 (1989)](https://www.cs.unc.edu/~stotts/COMP590-059-f24/robsrules.html) (Hacker News)
  这篇文档记录了 Rob Pike 在 1989 年提出的编程规则，虽然年代久远，但关于简洁性、数据驱动设计等原则在今天看来依然历久弥新。在 AI 辅助编程日益复杂的当下，重温这些基础原则有助于开发者回归软件工程的本质。
